feat: Add mobile PWA support with deployment infrastructure

Implemented complete Progressive Web App (PWA) support to enable mobile
deployment. Users can now install the app on their phones like a native
app without going through app stores.

## Features Added

### PWA Implementation
- Service Worker with offline support (web/sw.js)
- Web App Manifest for installable app (web/manifest.json)
- Mobile-first responsive UI updates
- Touch gestures (pull-to-refresh)
- Standalone mode (fullscreen without browser bar)
- App icons (192x192, 512x512)

### Deployment Infrastructure
- Fly.io configuration (fly.toml)
- Render.com configuration (render.yaml)
- Railway configuration (railway.json)
- Optimized production Dockerfile
- Docker build optimization (.dockerignore)
- GitHub Actions auto-deploy workflow

### Deployment Scripts
- deploy-mobile.sh (Unix) - Interactive deployment script
- deploy-mobile.bat (Windows) - Interactive deployment script
- test-pwa.sh (Unix) - PWA testing script
- test-pwa.bat (Windows) - PWA testing script
- Makefile targets for mobile development

## Technical Details

### PWA Features
- Offline support via service worker
- Cache-first strategy for static assets
- Network-first strategy for API calls
- Install prompt for home screen
- Standalone display mode
- Theme color and splash screen

### Deployment Options
- Fly.io: Global edge network, 3 VMs free
- Render.com: Auto-deploy from GitHub, 750 hours/month free
- Railway: Simple CLI deployment, $5 credit/month free

### Performance Targets
- First Contentful Paint: < 1.5s
- Time to Interactive: < 3.5s
- Lighthouse PWA Score: 100/100
- Bundle Size: < 500KB

## Testing

Run PWA tests:

## Deployment

Quick deploy:
```bash
make deploy-mobile
```

Or use platform-specific scripts:
```bash
./deploy-mobile.sh    # Linux/Mac
deploy-mobile.bat     # Windows
```
